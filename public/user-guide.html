üå≥ Task Hierarchy Management

Understanding WBS (Work Breakdown Structure)

Tasks are organized hierarchically using WBS numbers:

  • 1.0 - Top-level task (phase)
  • 1.1 - Subtask under 1.0
  • 1.1.1 - Sub-subtask under 1.1
  • 1.1.1.1 - Deepest level (4 levels max)
üì∏ Screenshot: Task hierarchy with different levels

Expand/Collapse Tasks

  • Click ‚ñ∂ arrow next to task to expand and show subtasks
  • Click ‚ñº arrow to collapse subtasks
  • Use ‚ñº button in header to expand all tasks at once
  • Use ‚ñ∂ button in header to collapse all tasks

Promote/Demote Tasks

Change task hierarchy levels using the colored circle icons on the left:

Icon Action Effect
üü¢ ‚óÄ Promote Move task left (up one level in hierarchy)
üîµ ‚ñ∂ Demote Move task right (down one level, becomes subtask)
üì∏ Screenshot: Promote/Demote icons with arrows
üí° Tip: Icons are grayed out when you've reached the maximum depth or can't move further. Task 1.0 cannot be promoted or demoted.

Drag and Drop Reordering

Reorder tasks at the same level by dragging:

  1. Click and hold on a task (except task 1.0)
  2. Drag to the desired position
  3. Green line shows where it will drop
  4. Release to complete the reorder
  5. WBS numbers update automatically
‚ö†Ô∏è Important: You can only reorder tasks within the same level. To move a task to a different level, use Promote/Demote.

üîó Managing Dependencies

What Are Dependencies?

Dependencies link tasks together so they happen in the correct order. For example: "Site Prep" can't start until "Permit Approval" finishes.

üì∏ Screenshot: Dependencies modal

Adding Dependencies

  1. Click "+ Add" or the dependency count in the Dependencies column
  2. Click "Add Dependency" button at the bottom
  3. Select which task must finish first (predecessor)
  4. Set dependency type (Finish-to-Start is most common)
  5. Add lag time if needed (e.g., 7 days for concrete curing)
  6. Click "Save Dependencies"

Dependency Types

Type Code Description Example
Finish-to-Start FS This task starts after predecessor finishes Foundation ‚Üí Framing

Lag Time

Lag time adds a delay between tasks:

  • 0 days: Start immediately when predecessor finishes
  • 7 days: Wait 7 days after predecessor finishes
  • Use case: Concrete curing, paint drying, approval processes

Automatic Date Cascading

üí° Magic Feature: When you change a task's dates, all dependent tasks automatically adjust! This is the power of dependencies.

Example:

  1. Task A ends on Jan 10
  2. Task B depends on Task A (FS, 2 days lag)
  3. Task B automatically starts on Jan 12
  4. If you move Task A to end on Jan 15, Task B automatically moves to Jan 17!

üîß Troubleshooting

Common Issues

Problem: Task dates won't change

Solution:

  • Check if task has dependencies - dependent tasks can't be moved freely
  • Verify date is within project start/end range
  • Ensure start date is before end date

Problem: Can't add subtask

Solution:

  • Check if you've reached maximum depth (4 levels)
  • Verify parent task is expanded
  • Try clicking the + icon directly on the task row

Problem: Dependencies not cascading

Solution:

  • Verify dependency is saved (check Dependencies column)
  • Ensure dependency type is Finish-to-Start
  • Check that predecessor task dates are set
  • Refresh the page if needed

Problem: AI generation failed

Solution:

  • Verify all required fields are filled
  • Check that start date is before end date
  • Ensure deliverables field is not empty
  • Try again - sometimes API can have temporary issues

Problem: Team members not showing in Workload

Solution:

  • Open Project Settings (hamburger menu ‚Üí Project Settings)
  • Verify team members are listed
  • Check that tasks are assigned to exact names (spelling matters)

Problem: Changes not saving

Solution:

  • Check header for "All changes saved ‚úì" message
  • Ensure you have internet connection
  • Press Enter to complete inline edits
  • Refresh page to verify persistence
